# ADR-0024: Offline Full-Matrix Compass Calibration

## 状态

Accepted

## 背景

MP157 Runtime 已支持三轴硬铁偏置和完整 3×3 软铁/安装矩阵，但真实整机仍未完成多姿态标定。History Recorder 已能保存 MMC5603 `magnetometer.csv`，当前缺少从原始采集数据生成候选参数、量化覆盖度和拒绝劣质采集的可复现工具。

标定工具用于开发主机离线处理，不属于每秒 Runtime 热路径。它必须避免把一次拟合结果自动标记为真实校准通过，并且不能假设磁力计数据能够唯一推导传感器安装方向。

## 可选方案

### 方案 A：每轴最小值/最大值校准

优点：

- 实现简单，容易解释。
- 能估计硬铁偏置和对角缩放。

缺点：

- 不能校正交叉轴软铁畸变。
- 极值对离群点敏感，不能满足 Runtime 已支持的完整 3×3 矩阵能力。

### 方案 B：使用 Python/NumPy/SciPy 椭球拟合

优点：

- 数值算法和可视化生态成熟。
- 原型开发快。

缺点：

- 增加 Python 包环境和版本管理。
- 当前仓库以 C/C++ 工程化和轻依赖为主，交付给面试官时需要额外环境说明。

### 方案 C：标准 C++17 实现归一化全椭球拟合

优点：

- 无第三方依赖，Windows/Linux 可使用同一 CMake 工程。
- 能输出完整 3×3 交叉轴校正，并可与 Runtime 配置直接对应。
- 可把覆盖度、残差、轴比和离群点处理纳入自动化测试。

缺点：

- 需要自行维护小矩阵求解和对称特征分解。
- 代数椭球拟合不是通用非线性最优估计，仍依赖充分、均匀的三维姿态覆盖。

## 最终决策

采用方案 C：

- 读取 History Recorder 的 nT CSV 并转换为 μT。
- 对样本做平移/尺度归一化，使用 9 参数完整二次曲面最小二乘拟合。
- 用 3×3 Jacobi 特征分解求正定平方根，生成全矩阵软铁校正。
- 把校正矩阵行列式归一为 1，保留三轴几何平均磁场尺度。
- 先用分量中位数/径向 MAD 排除极端强磁点，再按拟合残差稳健裁剪并重新拟合。
- 要求样本数、八分体、方向方差、轴比、RMS 和最大残差通过门槛。
- 可选接收独立测量的 proper sensor-to-body rotation，输出 `R * soft_iron`；不从磁力计椭球猜测安装方向。
- 候选配置始终写出 `compass_calibration_valid=false`，只有独立航向验收后才能人工启用。

## 决策理由

该方案把当前未完成的现场标定拆成“可复现候选拟合”和“真实航向验收”两个证据阶段。工具能够充分利用 Runtime 已有的完整矩阵接口，同时保持依赖成本和交叉平台构建简单，并用强制 false 防止把数学拟合误写成真实整机已经校准。

## 风险

- 姿态覆盖不均匀时，代数拟合中心可能有偏差；质量门槛只能拒绝明显不足，不能替代采集规范。
- 对称软铁校正只能确定到一个旋转自由度；安装方向必须独立测量。
- 固定在整机上的硬磁干扰会进入标定参数，环境临时磁干扰会污染结果，两者需要通过重复采集区分。
- 当前没有真实室外 CSV，合成畸变测试只能证明算法恢复能力和失败门槛，不能证明当前硬件校准精度。
