# ADR-0011: Use Bosch BMP3 Sensor API for BMP390

## 状态

Accepted

## 背景

BMP390 的原始温度和气压需要使用芯片内部校准参数进行补偿。该计算包含多项定点运算，直接自行重写会增加数值偏差和维护风险。

## 可选方案

### 方案 A：自行实现最小寄存器驱动和补偿公式

优点：

- 代码量可以按项目需要裁剪
- 完全控制接口和数据类型

缺点：

- 需要重新验证校准参数解析、溢出边界和补偿精度
- 后续对照数据手册和修复计算问题的成本较高

### 方案 B：集成 Bosch BMP3 Sensor API

优点：

- 使用 Bosch 提供并经过广泛使用的校准和补偿实现
- 同时支持 BMP388/BMP390，寄存器和状态处理完整
- BSD-3-Clause 许可证允许随固件源码集成

缺点：

- 引入约 4.8 KiB 的额外 Flash 使用量
- 需要通过项目适配层连接 HAL I2C 和延时接口

## 最终决策

集成参考工程中的 Bosch BMP3 Sensor API v2.0.5，并保持上游 `bmp3.c`、`bmp3.h` 和 `bmp3_defs.h` 独立放在 `firmware/third_party/bosch_bmp3/`。项目自有 `bmp390_provider_c` 负责 I2C、延时、地址探测和协议数据转换。

## 决策理由

BMP390 补偿算法的正确性比少量代码尺寸收益更重要。官方驱动不需要动态内存或操作系统依赖，能直接使用 GNU Arm Embedded Toolchain 交叉编译，适合当前 bare-metal 固件。

## 风险

- 真实模块的 I2C 地址、供电、上拉和数据就绪行为尚未上板确认。
- 上游驱动版本后续升级时，需要重新执行构建和真实传感器回归验证。
