# ADR-0004: 实现最小 NMEA Parser

## 状态

Accepted

## 背景

Stage 0.4 需要从 NMEA 文件回放推进到基础 GNSS 定位状态输出。当前项目仍处于 Linux 用户态 Runtime 原型阶段，尚未接入 UBLOX-M10 串口，也不需要完整覆盖所有 NMEA 语句。

## 可选方案

### 方案 A: 引入第三方 NMEA 解析库

优点：

- 协议覆盖更完整。
- 可以减少自研解析代码。

缺点：

- 增加交叉编译和嵌入式部署依赖。
- 当前阶段需求很小，依赖成本高于收益。
- 不利于展示基础 GNSS 数据处理思路。

### 方案 B: 使用 C++17 标准库实现最小 Parser

优点：

- 不引入第三方依赖。
- 只覆盖当前 Stage 0 需要的 RMC/GGA 基础字段。
- 便于后续逐步扩展和测试。

缺点：

- 当前不覆盖完整 NMEA 协议。
- 当前不做 checksum 校验。
- 需要后续补充更多测试样例。

## 最终决策

Stage 0.4 使用 C++17 标准库实现最小 NMEA Parser，当前只解析 RMC 和 GGA 语句中的基础定位状态。

## 决策理由

RMC 和 GGA 已能支撑当前阶段的基础定位状态输出，包括有效定位、经纬度、速度、海拔、卫星数和 fix quality。该方案保持依赖轻量，符合 Stage 0 的原型边界。

## 风险

- 当前未校验 NMEA checksum，不能用于高可靠数据判定。
- 当前字段覆盖有限，无法替代完整 GNSS 数据处理模块。
- 当前只基于样例数据验证，真实 UBLOX-M10 输出接入后可能需要增强解析。
