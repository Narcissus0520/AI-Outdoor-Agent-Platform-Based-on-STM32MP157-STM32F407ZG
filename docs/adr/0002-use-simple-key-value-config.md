# ADR-0002: 使用简单 key=value 配置格式

## 状态

Accepted

## 背景

Stage 0.2 需要增加基础配置文件加载能力，用于配置默认 NMEA 输入路径和日志级别。当前项目仍处于 Linux 用户态 Runtime 原型阶段，目标是保持实现轻量、易懂、可交叉编译，不希望过早引入 JSON/YAML/TOML 等第三方依赖。

## 可选方案

### 方案 A: 简单 key=value 文本配置

优点：

- 不需要第三方依赖。
- 实现和调试成本低。
- 对当前 `nmea_input_path`、`log_level` 等少量配置项足够。

缺点：

- 不适合复杂嵌套配置。
- 需要项目自行维护解析和错误提示。

### 方案 B: JSON/YAML/TOML 配置

优点：

- 格式表达能力更强。
- 更适合复杂配置结构。

缺点：

- 通常需要引入第三方库。
- 会增加交叉编译和嵌入式部署维护成本。
- 对 Stage 0.2 的需求偏重。

## 最终决策

Stage 0.2 使用简单 `key=value` 文本配置格式，配置文件放在 `config/runtime.conf`。

## 决策理由

当前阶段只需要配置 NMEA 输入路径和日志级别，简单文本配置可以满足需求，并保持 C++17 标准库实现。后续如果配置复杂度明显上升，再评估是否迁移到更结构化的格式。

## 风险

- 配置项增多后，简单格式可能不够表达复杂模块关系。
- 当前解析器只支持基础行格式、注释和空白裁剪。
